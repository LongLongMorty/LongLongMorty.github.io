Cpp实现单例模式

发表于2026-02-05|更新于2026-02-05|设计模式
|浏览量:

在 C++ 中，单例模式（Singleton Pattern）是最简单但也最容易写出 Bug 的设计模式。它的核心目标是：确保一个类只有一个实例，并提供一个全局访问点。

让我们从最基础的”初学者版本”开始，逐步演进到工业级的实现。

1. 基础版：懒汉式（线程不安全）

这个版本最容易想到，只有在第一次调用 getInstance() 时才创建实例。

缺点：在多线程环境下，如果两个线程同时判断 instance == nullptr，可能会创建出两个实例，导致内存泄漏和逻辑错误。

2. 进阶版：饿汉式（天然线程安全）

在程序启动时（进入 main 函数前）就完成初始化。

优点：绝对的线程安全。

缺点：”不环保”。如果这个类很大且一直没被用到，它也会占用内存。此外，如果有多个单例类互相依赖，它们的初始化顺序在 C++ 中是不确定的。

3. 工业级懒汉式：Meyers’ Singleton（推荐）

这是 C++11 之后最推荐、最简洁的写法，由《Effective C++》作者 Scott Meyers 提出。它利用了 C++11 标准中的一个特性：局部静态变量的初始化是线程安全的。

要理解线程安全的本质，需要知道 C++11 标准对「局部静态变量初始化」的明确要求：

初始化时机：局部静态变量的初始化发生在「第一次执行到该变量定义的代码行」时，而非程序启动时（这也是”懒汉式”的核心）。

线程安全保障：编译器会自动为局部静态变量的初始化过程添加线程同步机制（比如隐式的互斥锁），确保：

  • 只有一个线程能执行初始化操作；
  • 其他线程会被阻塞，直到初始化完成；
  • 初始化完成后，所有线程都只会直接访问已初始化的变量，不再触发初始化逻辑。
